Metadata documentation is valuable when there is a need to determine if the data required for a new system, profiling effort, or algorithm is both available and appropriate to the task. For example, the proposed requirements for patient demographic data matching algorithms include, at a minimum, the following:

  • Given Name
  • Last Name
  • Date of Birth
  • Administrative Gender

Metadata should be clearly described and stored in a location that is easily accessible for all relevant stakeholders. Defining and populating metadata is a task that has few dependencies. It can be conducted in manageable phases, either standalone or associated with new development efforts, depending on the resources and needs of the organization.

Example Work Products

  • Metadata compendium or virtual metadata repository (single or multiple sources)
